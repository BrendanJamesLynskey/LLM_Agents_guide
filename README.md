# How LLM Agents Work — A Visual Deep Dive

A comprehensive, interactive visual guide that breaks down every operation inside an LLM-powered agent — from tokenization to tool execution to memory management.

## Live Demo

**https://brendanjameslynskey.github.io/LLM_Agents_guide/**

## Overview

This single-page web application provides a detailed visual walkthrough of LLM agent architecture, covering:

- **Agent Anatomy** — The five core components: LLM Core, Tools, Planner, Memory, and Orchestrator
- **The Agent Loop** — The Perceive → Reason → Plan → Act → Observe → Update execution cycle
- **Prompt Assembly** — How system prompts, tool definitions, and conversation history are constructed
- **Reasoning Engine** — Chain-of-Thought, ReAct, Tree-of-Thought, and Reflexion patterns
- **Tool Use** — Step-by-step mechanics of how agents call external tools
- **Memory Systems** — Working memory, short-term scratchpad, and long-term vector stores (RAG)
- **Planning** — Task decomposition, dependency analysis, and adaptive replanning
- **Agent Patterns** — ReAct, Plan-and-Execute, LLM Compiler, Router, and Self-Reflective architectures
- **Multi-Agent Systems** — Hierarchical, peer-to-peer, and pipeline coordination topologies
- **Interactive Demo** — Live execution traces for weather queries, research tasks, code generation, and multi-step workflows
